Glia

A multiplayer visual programming environment for collaborative music creation. Built on CMajor and JUCE, it enables real-time collaborative audio graph editing—I took it on the road for successful UK shows in December 2025.

Listen / watch

What it is

Glia is a collaborative project building infrastructure for collaborative music creation. It's not just a plugin—it's a complete visual programming environment where multiple users can build and edit audio processing graphs simultaneously.

Think: Google Docs for modular synthesis. Drag-and-drop nodes, real-time code generation, WebSocket synchronization, and conflict-free collaborative editing via CRDTs. Runs as a VST3/AU plugin, standalone app, or web client.

I didn't build Glia, but I'm a contributor and early adopter—integrating it into my performance workflow and stress-testing it in live contexts.

Technical stack

CMajor audio engine

High-performance audio DSL compiled to native code or WebAssembly for real-time processing.

Visual graph editor

React Flow-based interface for building audio graphs—nodes, connections, groups, MIDI mappings.

Real-time collaboration

Yjs CRDTs enable conflict-free simultaneous editing. Cursor tracking, viewport sync, offline-capable.

How I use it

Live performance

Runs as a VST3 in Ableton Live—custom audio graphs responding to MIDI controllers and audio input.

Rig integration

Sits alongside hardware synths, guitar, field recordings. Designed for reliability and fast troubleshooting on stage.

Visual mapping (next)

Developing VDMX-based workflows for audio-reactive visuals driven by Glia's output.
